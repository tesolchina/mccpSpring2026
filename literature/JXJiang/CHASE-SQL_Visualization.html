Section 1: Introduction ~1000 words
Move 1: Establishing Territory
Centrality Claims: Text-to-SQL is "crucial" for many use cases, bridging human language and SQL
"Text-to-SQL, as a bridge between human language and machine-readable structured query languages, is crucial for many use cases, converting natural language questions into executable SQL commands."
Strong centrality claims supported by practical applications. Connects technical NLP work to real-world database interactions.
Move 2: Establishing Niche
Gaps Identified: Self-consistency methods ineffective (14% gap), single prompt design limitations
"In fact, as illustrated in Table 1, the most consistent answers would not always be the correct ones, with an upper-bound performance 14% higher than that achieved through self-consistency. This substantial gap highlights the potential for significant improvement by implementing more effective selection methods."
Specific, quantitative gap identification (14% gap) demonstrates the problem clearly with empirical evidence.
Move 3: Occupying Niche
Solution: CHASE-SQL framework with three candidate generation methods and selection agent
"Specifically, we propose three distinct candidate generation approaches, each capable of producing high-quality responses. The first is inspired by the divide-and-conquer algorithm... The second employs a query execution-plan-based chain-of-thought strategy... Lastly, we introduce a novel online synthetic example generation method."
Clear positioning with enumerated contributions. Each method has specific technical description and motivation.

Section 2: Methods ~4000 words
2.1 Overall Framework
Components: Four primary components: Value retrieval, Candidate generator, Query fixer, Selection agent
"This section outlines the proposed CHASE-SQL framework, which consists of four primary components: 1) Value retrieval, 2) Candidate generator, 3) Query fixer, and 4) Selection agent."
Clear modular organization with enumerated components. Strong technical structure appropriate for CS/NLP audience.
2.3 Multi-Path Candidate Generation
Three Methods: Divide-and-Conquer CoT, Query Plan CoT, Online Synthetic Example Generation
"Divide and Conquer CoT: Divide-and-conquer perspective brings breaking down complex problems into smaller sub-problems, solving each individually, and then combining the solutions to obtain the final answer."
Excellent conceptual framing - connects classical algorithm design (divide-and-conquer) to modern LLM prompting.
2.5 Selection Agent
Approach: Pairwise comparison-based selection using fine-tuned binary classification model
"Given a set of candidates SQL queries C = {c1, c2, ..., cn}, the final responses are selected by finding the candidate that has the highest score assigned by the selection model. This model θp can take k candidates and rank them based on how accurately each of them answers the given question."
Clear mathematical formulation with formal notation. Notice the use of θp for the selection model, maintaining consistent mathematical notation.

Imitation Framework for Future Papers

Structural Elements to Adapt:

  • Multi-Component Framework: Breaking complex systems into well-defined modules (Value Retrieval → Candidate Generation → Query Fixer → Selection Agent)
  • Algorithmic Descriptions: Formal algorithm presentations (Algorithm 1, 2, 3) with clear specifications
  • Ablation Studies: Systematic component-wise evaluation demonstrating necessity of each part
  • Multi-Dataset Evaluation: Testing generalizability across different benchmark datasets

Rhetorical Strategies:

  • Gap Identification through Empirical Evidence: Using quantitative gaps (e.g., "14% higher upper bound") to demonstrate problems
  • Contribution Enumeration: Clear listing of technical contributions with specific technical descriptions
  • State-of-the-Art Positioning: Strong claims backed by quantitative results and benchmark rankings
  • Framework Justification: Connecting technical design choices to identified limitations

Quality Indicators:

  • Mathematical Rigor: Formal notation, algorithmic descriptions, mathematical formulations
  • Empirical Validation: Multiple evaluation metrics, ablation studies, comprehensive comparisons
  • Modular Design: Clear component separation with systematic evaluation
  • Reproducibility: Detailed algorithmic descriptions, hyperparameter specifications
