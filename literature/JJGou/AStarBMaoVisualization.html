Section 1: Introduction (with Integrated Related Work) ~3 pages
Move 1: Establishing Territory
Centrality Claims: Graph similarity search is "fundamental problem" in graph databases
"RETRIEVING all occurrences of a query graph in a graph database is a fundamental problem in graph database research... In many applications, searching for the exact occurrences of a query graph may return no result or very few results that are not sufficient for the applications."
Strong opening establishing fundamental problem and practical motivation. Progression from exact matching to similarity search justifies research direction.
Move 2: Establishing Niche
Specific Gap: AStar-LSa memory consumption problem
Context: When graphs/thresholds become larger
"We observe that the main memory consumption of AStar-LSa increases very fast when either the graph size or the threshold τ becomes larger. This limits AStar-LSa from scaling to larger graphs or larger threshold values. Moreover, the main memory consumption... is inversely related to the tightness of the lower bound estimation"
Highly specific gap: not correctness but scalability. Provides theoretical insight connecting memory to lower bound tightness, motivating the solution.
Move 3: Occupying Niche - Two-Stage Solution
Stage 1: lbBMa (tighter, O((|V(q)| + |V(g)|)⁴))
Stage 2: lbBMao (efficient, O((|V(q)| + |V(g)|)³))
"We in this paper first propose an anchor-aware branch match-based lower bound lbBMa... However, due to the high time complexity... we then slightly loose the lower bound lbBMa into lbBMao such that... can be computed in O((|V(q)| + |V(g)|)³) total time."
Sophisticated two-stage contribution showing iterative refinement. Explicitly discusses trade-off: ideal solution → practical solution. Three enumerated contributions.
Integrated Related Work
Four Themes: (1) Graph Similarity Search, (2) GED Computation, (3) Maximum Common Subgraph, (4) ML Methods
Related work integrated into introduction rather than separate section. Each theme ends with positioning statement. Dismisses irrelevant approaches clearly.
↓
Section 2: Preliminaries ~1.5 pages
Formal Definitions
Mathematical Foundation: Graph definition, GED definition, problem statement
Notation Table: Table 1 summarizes frequently used notations
"Definition 2.1: The graph edit distance (GED) between two graphs q and g, denoted ged(q, g), is the minimum number of edit operations that can transform q into g."
Clean mathematical formulation. Table 1 (notation summary) enhances clarity. Simplifying assumptions clearly stated with justifications.
↓
Section 3: State-of-the-Art Approach AStar-LSa (BASELINE) ~2.5 pages
Comprehensive Baseline Review
Coverage: Search tree structure, mapping enumeration, lower bound lbLSa
Purpose: Establishes framework to be improved
"The state-of-the-art approach AStar-LSa [1] computes ged(q, g) by enumerating vertex mappings from V(q) to V(g)... The efficiency of AStar-LSa mainly comes from two aspects. Firstly, AStar-LSa designed the anchor-aware label set-based lower bound lbLSa which is much tighter than the label set-based lower bound lbLS..."
Full section dedicated to baseline - unusual but effective. Explains both strengths (tighter bounds) and limitations (memory consumption). Sets up framework for refinement.
↓
Section 4: Tighter Lower Bound lbBMa (IDEAL SOLUTION) ~3 pages
Branch Match-Based Lower Bound
Key Innovation: Branch matching instead of label set matching
Formal Guarantees: Correctness proof + Tightness proof (lbBMa ≥ lbLSa)
Complexity: O((|V(q)| + |V(g)|)⁴)
"We in this section propose a new lower bound lbBMa that is based on branch matching instead of label set matching... We formally prove that lbBMa is tighter than lbLSa."
Rigorous theoretical development: intuition → formal definition → correctness proof → tightness proof → algorithm → complexity. Exemplary structure for algorithmic contribution.
⚠️ TRADE-OFF IDENTIFIED: lbBMa is tighter but O((|V(q)| + |V(g)|)⁴) is too slow in practice
↓ OPTIMIZATION REFINEMENT ↓
Section 5: Trading Tightness for Efficiency (PRACTICAL SOLUTION) ~3 pages
Optimized Lower Bound lbBMao
Strategy: Slightly relax lbBMa for better complexity
Maintained Guarantee: Still tighter than lbLSa (lbBMao ≥ lbLSa)
Improved Complexity: O((|V(q)| + |V(g)|)³)
"To strike a balance between the tightness and the efficiency of lower bound estimation, we then slightly loose the lower bound lbBMa into lbBMao such that the lower bound costs of all children of f regarding lbBMao can be computed in O((|V(q)| + |V(g)|)³) total time."
Sophisticated optimization showing iterative refinement. Explicit trade-off articulation: "strike a balance." Maintains key guarantee (tighter than baseline) while improving efficiency. This is the practical contribution.
✓ OPTIMAL BALANCE: lbBMao achieves best practical trade-off between tightness and efficiency
↓
Section 6: Experiments ~3 pages
Multi-Dimensional Evaluation
Metrics:
• Search space size (validates tightness claims)
• Memory consumption (addresses main motivation)
• Running time (overall efficiency)
• Scalability (varies graph size and threshold)
"Empirical studies on real datasets demonstrate that our newly proposed algorithm AStar-BMao runs faster, and at the same time consumes much less main memory, than AStar-LSa."
Comprehensive evaluation validates both theoretical claims (tightness) and practical benefits (memory, speed). Shows AStar-BMa has smallest search space but AStar-BMao achieves best overall performance. Comparison to ML methods shows orders of magnitude improvement.
↓
Section 7: Conclusion ~0.5 pages
Synthesis & Impact
Key Achievement: Faster + less memory than state-of-the-art
Dual Benefit: Emphasizes both speed AND memory improvements
Concise conclusion reinforcing main message. Emphasizes dual benefit rather than single metric improvement.

Incremental Innovation Model

Two-Stage Refinement Strategy:

Stage 1 (lbBMa): Optimize for tightness → Smallest search space but O(n⁴) complexity

Stage 2 (lbBMao): Optimize for efficiency → Slightly larger search space but O(n³) complexity

Result: lbBMao achieves best practical balance

This two-stage narrative is pedagogically effective and demonstrates sophisticated algorithm design through iterative refinement.

Imitation Framework for Future Papers

Structural Elements to Adapt:

  • Two-Stage Contribution: Present ideal solution then practical optimization
  • Baseline Deep Dive: Dedicate full section to explaining state-of-the-art
  • Progressive Refinement: Show evolution of ideas clearly
  • Notation Table: Include table of frequently used notations for clarity
  • Integrated Related Work: Can place in introduction for journal papers

Rhetorical Strategies:

  • Gap via Scalability: "Limits... from scaling to larger..." identifies practical limitation
  • Formal Guarantees: "We formally prove that..." emphasizes rigor
  • Trade-off Framing: "To strike a balance between..." shows sophistication
  • Comparative Positioning: Constant comparison to state-of-the-art throughout
  • Quantified Improvements: "Orders of magnitude faster" makes concrete claims

Quality Indicators:

  • Theoretical Rigor: Correctness proofs, tightness proofs, complexity analysis
  • Practical Impact: Addresses real limitation (memory consumption)
  • Multi-Dimensional Evaluation: Time, space, search space metrics
  • Scalability Analysis: Systematically varies problem parameters
  • Clear Progression: Evolution of ideas clearly traced (lbLS → lbLSa → lbBMa → lbBMao)
  • Honest Trade-offs: Acknowledges when ideal solution is impractical
