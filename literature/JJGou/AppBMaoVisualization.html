Section 1: Introduction ~3 pages
Move 1: Establishing Territory
Centrality Claims: GED is "fundamental metric" with wide applications
"Graph edit distance (GED) is a fundamental metric for measuring the distance or similarity between two graphs. It has been widely used in various applications including molecular chemistry, pattern recognition, graph database, computer vision and biology."
Strong opening with "fundamental" + diverse application domains. Establishes NP-hardness as key challenge.
Move 2: Establishing Niche
Two Significant Limitations: (1) Limited cross-field evaluation, (2) Weak baseline comparisons
"The existing studies on learning-based methods for GED prediction have two significant limitations. (1) The development of deep learning models... has been explored in various research fields... yet cross-field evaluations have been quite limited. (2) More importantly, all these advancements have been evaluated against a simple combinatorial heuristic baseline..."
Highly specific gap identification with enumeration. Figure 1 provides visual evidence of cross-field gaps. "More importantly" creates hierarchy.
Move 3: Occupying Niche
Dual Contribution: (1) Holistic survey with taxonomy, (2) New heuristic App-BMao
"We first conduct a holistic review of the existing learning-based methods, categorizing them into non-interpretable and interpretable GED prediction approaches... Secondly, we present a simple yet effective combinatorial heuristic algorithm App-BMao for GED estimation..."
Clear two-pronged contribution. Complexity analysis provided: O(t × (|V(q)| + |V(g)|)³). Three enumerated key findings and three main contributions.
↓
Section 2: Preliminaries ~2 pages
Formal Definitions
Mathematical Foundation: Graph definition, isomorphism, edit path, GED
"Definition 2.3. Given two graphs q and g, the graph edit distance (abbreviated as GED) between q and g is defined as the minimum length among all possible edit paths between q and g"
Rigorous mathematical formulation with running example (Figure 2). Clear progression from basic to complex concepts.
↓
Section 3: Exact GED Computation ~2 pages
State-of-the-Art Review
Foundation: Reviews exact algorithms (A*GED, DF-GED, AStar-LSa, AStar-BMao)
"The state-of-the-art algorithms, AStar-LSa [11] and AStar-BMao [12], again used best-first search, but proposed much tighter lower bounds than the previous ones by formulating the idea of anchor-aware lower bounds."
Establishes algorithmic framework (search tree, node mappings) that learning-based methods will adapt. Sets up comparison baseline.
↓
Section 4: Learning-based Approaches for GED Prediction (SURVEY) ~8 pages
4.1 Graph Neural Networks
Foundation: GNN basics - message passing, node embeddings, graph pooling
Pedagogical introduction to GNN for readers unfamiliar with deep learning.
4.2 Non-interpretable GED Prediction
Framework Identification: Three-module framework (Siamese GNN → Embedding Fusion → Prediction Head)
Models: SimGNN, GraphSim, GENN, H2MN, TaGSim, EGSC, ERIC, OMC, GREED
"We show that these approaches share a general three-module framework... Module-I is a Siamese GNN that separately maps the two input graphs into both node-level embeddings and graph-level embeddings. Module-II fuses together those embeddings... Module-III... making the final GED prediction."
Major contribution: abstracts common design pattern across different models. Table 1 systematically compares all approaches. Shows deep understanding through framework identification.
4.3 Interpretable GED Prediction
Two Categories: (1) Tree traversal-based (GENN-A*, Noah), (2) Cost matrix-based (GOTSim, GEDGNN)
"Learning-based approaches for interpretable GED prediction outputs an edit path (more precisely, a node mapping) along with its cost as the estimated GED value. The outputted edit path serves as an evidence of the estimated GED value, and thus makes the prediction interpretable."
Further taxonomy within interpretable methods. Connects back to exact algorithms (Section 3) showing how interpretable methods bridge learning and traditional approaches.
↓
Section 5: App-BMao Algorithm (CORE CONTRIBUTION) ~3 pages
New Heuristic Algorithm
Key Innovation: Adapts exact algorithm AStar-BMao for estimation
Controllable Complexity: Parameter t controls time/space trade-off
Guarantees: Accuracy improves with increasing t
"We present a simple yet effective combinatorial heuristic algorithm App-BMao for interpretable GED estimation, which is adapted from the existing exact GED computation algorithm of [12]. App-BMao takes an input parameter t for controlling the time and space complexity. The time complexity of App-BMao... is O(t × (|V(q)| + |V(g)|)³), and its space complexity is O(t + |V(q)| × |V(g)|)"
Clear algorithmic contribution with explicit complexity bounds. Emphasis on "interpretable" and "controlled complexity" as advantages over learning-based methods.
↓
Section 6: Experiments ~4 pages
Comprehensive Evaluation
Scope: Against ALL surveyed learning-based methods
Datasets: Three widely used datasets
Metrics: Estimation accuracy, running time, memory consumption
Key Finding: App-BMao outperforms all learning-based approaches
"Extensive empirical evaluations on three widely used datasets show that the new heuristic algorithm App-BMao outperforms all existing learning-based approaches for both interpretable and non-interpretable GED prediction."
Rigorous evaluation validates survey's critical observations. Experiments structured to answer research questions from introduction. Compares against EVERY method surveyed, not just selected baselines.
↓
Section 7: Related Work ~1 page
Broader Context
Coverage: Other GED-related work not covered in Section 4 survey
Positioned late because Section 4 already comprehensively covered learning-based methods. This section covers approximate algorithms, other similarity measures, etc.
↓
Section 8: Conclusion ~0.5 pages
Synthesis & Impact
Key Message: Simple heuristics can outperform complex learning-based methods
Contributions: Survey + algorithm + comprehensive evaluation
Strong conclusion reinforcing counter-narrative: sometimes traditional approaches outperform deep learning when properly designed.

Imitation Framework for Future Papers

Structural Elements to Adapt:

  • Survey-as-Contribution: Make comprehensive literature review a primary contribution
  • Taxonomy Development: Create original categorization schemes for existing work
  • Framework Abstraction: Identify common design patterns across different approaches
  • Dual Contribution: Combine survey with new method for stronger paper
  • Late Related Work: Place broader related work after experiments if comprehensive survey earlier

Rhetorical Strategies:

  • Enumerated Gaps: "(1)... (2)..." creates clear, specific limitations
  • Visual Argumentation: Use figures to support claims (Figure 1 shows cross-field gaps)
  • Critical Survey: Don't just describe - analyze and critique existing work
  • Counter-Narrative: Challenge conventional wisdom (heuristic beats deep learning)
  • Hierarchy Creation: "More importantly..." to prioritize among multiple points

Quality Indicators:

  • Cross-Community Synthesis: Covers DB, ML, IR, CV communities
  • Systematic Comparison: Table 1 compares all approaches systematically
  • Comprehensive Evaluation: Tests against every surveyed method
  • Complexity Analysis: Explicit time and space complexity bounds
  • Balanced Perspective: Acknowledges when learning methods work well vs. when heuristic wins
