Section 3: Methods ~2000 words
Foundation Model Design
Input features, target-aware HSTU, MTML loss; formal mathematical definitions
Expert Design
FM Embedding Module, FM Fusion Module, lightweight HSTU; 20-40% compute reduction
System Deployment (HyperCast)
Decoupled training, high freshness (minutes), multi-tier inference
"Building upon the original HSTU architecture, we introduce an architectural simplification: instead of interleaving item and action embeddings, we combine them via direct summation. Furthermore, to prevent label leakage from user history, we remove the auto-regressive auxiliary losses. This optimization effectively halves the input sequence length, yielding a 50% reduction in complexity for the linear projection layers and a 25% reduction for the attention operations."
Analysis: Highly technical with quantitative efficiency claims. "Label leakage" is important production concern.
↓
Section 4: Experiments ~1800 words
Experiment Setup
Normalized Entropy (NE) metric; formal mathematical definition
Target-Aware Embedding Effectiveness
Ablation vs. user embeddings; substantial NE improvements
Transfer Ratio
Novel metric: TR in range [0.64, 1.0]; quantifies FM→Expert efficiency
Online Performance
A/B test results; statistically significant improvements; neutral latency
"We define the Transfer Ratio (TR) between a pair of FMs for a given expert as, TR = (NE(Expert_FM₁) - NE(Expert_FM₂)) / (NE(FM₁) - NE(FM₂)). A higher TR value signifies a more efficient paradigm, ensuring that investments in scaling the FM yield corresponding performance gains in downstream expert models."
Analysis: Novel metric with clear formal definition. TR in [0.64, 1.0] is quantitatively stated. Production A/B validation.
↓

Analysis: Concrete scale evidence emphasized. "Proven blueprint" positioning is strong claim.
